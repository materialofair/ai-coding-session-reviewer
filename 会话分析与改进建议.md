## ⚠️ 需要改进的习惯

### 1. **过度使用"继续"指令**
**问题**: 在23条用户消息中，有5次使用"继续"或类似指令（占比21.7%）

**影响**:
- 可能表示任务没有一次性完成
- 增加了会话轮次
- 降低了效率

**改进建议**:
```
❌ 不好的做法:
User: "帮我实现功能A"
Assistant: [部分实现]
User: "继续"
Assistant: [继续实现]
User: "继续生成"

✅ 更好的做法:
User: "帮我实现功能A，请一次性完成以下步骤：
1. 创建组件结构
2. 实现核心逻辑
3. 添加样式
4. 编写测试
请使用 /autopilot 或 /ralph 模式确保完整执行"
```

**具体行动**:
- 使用 `/autopilot` 进行自主完整执行
- 使用 `/ralph` 进行持续循环直到完成
- 使用 `/ultrawork` 进行并行任务执行

### 2. **简单问候浪费会话**
**问题**: 检测到2次"你好"类型的简单问候

**影响**:
- 浪费了会话轮次
- 没有实际价值

**改进建议**:
```
❌ 避免:
User: "你好"
Assistant: "你好！有什么我可以帮助你的吗？"
User: "帮我分析..."

✅ 直接进入主题:
User: "帮我分析当前项目的聊天记录，找出重复提问的问题"
```

### 3. **文件路径错误频繁**
**问题**: 12次错误中有2次是"File does not exist"

**原因分析**:
- 可能是工作目录切换导致
- 可能是路径拼写错误
- 可能是文件尚未创建

**改进建议**:
```
✅ 在请求操作文件前，先确认：
1. 当前工作目录: pwd
2. 文件是否存在: ls -la <path>
3. 使用绝对路径而非相对路径

示例:
User: "请先确认 .omc/conductor/spec.md 是否存在，
如果不存在请先创建，然后编辑它添加以下内容..."
```

### 4. **缺少明确的完成标准**
**问题**: 从"继续"指令的频繁使用可以看出，任务完成标准不够明确

**改进建议**:
```
✅ 在提问时包含完成标准:
User: "帮我实现AI助手面板功能，完成标准：
1. ✅ 右侧面板UI完成
2. ✅ 三个AI服务集成完成
3. ✅ 聊天记录总结功能可用
4. ✅ 所有测试通过
5. ✅ 文档更新完成

请使用 /ralph 模式确保所有标准都达成"
```

### 5. **分析/总结请求过于笼统**
**问题**: 3次分析/总结请求，但缺少具体指标

**示例**:
- ❌ "帮我分析当前项目"
- ❌ "总结一下"

**改进建议**:
```
✅ 具体化分析请求:
User: "帮我分析当前项目的会话记录，重点关注：
1. 我最常遇到的3类问题
2. 哪些问题我反复提问超过2次
3. 哪些问题始终没有得到解决
4. 我的提问方式有哪些可以改进的地方
5. 生成一份Markdown报告保存到 ./analysis-report.md"
```
